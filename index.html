In retrospect
  • Mosaic, the first popular graphical Web browser
  • Netscape Communications realized that the Web needed to become more dynamic, 1993
  • They decided to develop a scripting language, easy to use, where the code could be written directly in the Web page markup
  • Brenden Eich
In retrospect
  • Later in 96, JScript, a reverse-engineered implementation of Netscape's JavaScript, was part of Internet Explorer 3
  • Internet Explorer 3 also included Microsoft's first support for CSS and various extensions         to HTML, but in each case the implementation was noticeably different to that found in         Netscape Navigator at the time
  • Browser wars. Standardization was needed
  • Netscape submitted JavaScript to Ecma International to carve out a standard specification
In retrospect
  • Ajax, Jesse James
  • web applications where data can be loaded in the background, avoiding the need for full         page reloads and leading to more dynamic applications
  • Prototype, jQuery, Dojo Toolkit, MooTools
  • SPA. Angular, Backbone, React , Ember ...

Angular

Open Source Javascript Framework

  • get rid of difficulty in maintaining initial codebase
  • Code Production became fast
  • Developed by Google
  • Easily testable
  • Actively supported by Communities

Key Features

  • easy data binding,two way data binding

    Name:
    Hello {{input}}

  • Reusable Components, Directives
  • MVC Design pattern
  • Routing
  • Templating

How

  • Scope is the glue between application controller and the view
  • As the scope grows affects performance
  • Digest Cycle, Dirty Checking
  • Slow performance
  • Tightly coupled entities like controllers, views, services
  • EvolvingWeb

Angular2

  • Everything is a Component
  • No Scope, digest cycle No controller in Angular2
  • Better Dependency Injection through Constructor
  • Loosely coupled
  • Typescript
  • Faster

Differences

Angular1

2.more choice for languages,ES5,ES6,Dart,TypeScript

ng-repeat="technology in technologies"

ng-if="technologies.length"

7. Property binding ng-bind

8. Event binding ng-click

button ng-click="doSomething()"

9. ng-model two way data binding

input ng-model="technology.name"

10. Can define Service in 5 different ways
11. Use $injector service for Dependency Injection
Angular2
1.mobile oriented.
2. ES5, ES6, Dart
3.controllers are replaced with "Components"
4.Only way to bootstrap

import { bootstrap } from 'angular2/platform/browser'; import { ProductComponent } from './product.component'; bootstrap(ProductComponent);

5. *ngFor

*ngFor="let technology of technologies"

6. *ngIf

*ngIf="technologies.length"

7. One-way data -binding using[]

[value]="technology.name", [style.color]="color"

8. (ngClick)

button (click)="doSomething()"

9. ([ngModel]) two way data binding

input [(ngModel)]="technology.name"

10. class is the only way to define a service.
11. everything is class in Angular, so DI is achieved via constructor.

Directives

In Angular 1 we had the DDO (directive definition objects) which we used to develop our directive.

Confusing with transclusion , link , complie , scope and all

ng2 Sample Directive

Attribute Directives: directives that change the behavior of a component or element but don't affect the template ngStyle, ngClass

Structural Directives: directives that change the behavior of a component or element by affecting how the template is rendered ngIf, ngFor, ngSwitch
